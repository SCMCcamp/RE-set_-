================================================================================
RE:Set 留言墙 —— 温暖、复古的社区留言板
================================================================================

✨ 一个开箱即用的留言墙系统，支持便利贴白板墙、评论回复、匿名/实名私信以及管理员后台。
   数据实时同步，已预配置 Supabase 后端，部署即可使用。

--------------------------------------------------------------------------------
📸 功能特性
--------------------------------------------------------------------------------

【公开留言墙】
  • 所有人可见，支持回复评论
  • 分页加载（每页 10 条），性能友好
  • 可删除自己的留言（管理员可删除任何留言）

【私信聊天】
  • 联系人列表自动维护
  • 支持匿名发送（对方仅看到“匿名”）
  • 消息气泡区分发送/接收，显示时间戳
  • 可删除自己发出的消息

【用户系统】
  • 昵称唯一性（由 Supabase 保障）
  • 昵称设置后不可自行修改（管理员可协助）

【管理员功能】
  • 登录密码：reset2024（可在代码中修改）
  • 修改任意用户昵称
  • 删除任意用户及其所有数据（留言、私信）
  • 查看所有注册用户列表

【实时同步】
  • 基于 Supabase Realtime，无需手动刷新
  • 新留言、新评论、新私信即时推送

【交互体验】
  • 自定义模态框提示（非原生 alert）
  • 删除操作二次确认（带 ⚠️ 图标）
  • 手机端按钮按压反馈

🚀 部署指南（零配置）
--------------------------------------------------------------------------------

方法一：GitHub Pages（推荐）

  1. 下载本仓库的 index.html 文件
  2. 上传到您的 GitHub 仓库根目录
  3. 进入仓库 Settings → Pages → Source 选择 main 分支 → 保存
  4. 等待 1-2 分钟，访问 https://你的用户名.github.io/仓库名/

✅ 本项目已内置 Supabase 连接信息，开箱即用，无需任何环境变量配置。

方法二：本地测试

  直接双击 index.html 用浏览器打开即可（需联网加载 Supabase SDK）。

--------------------------------------------------------------------------------
🔐 管理员使用
--------------------------------------------------------------------------------

1. 登录管理员
   点击右上角用户头像 → “管理员登录” → 输入密码 reset2024（默认）

2. 管理用户
   • 登录后出现“修改昵称”“用户列表”按钮
   • 在“用户列表”中可删除任意用户及其全部留言/私信
   • 在私信聊天界面右上角也可直接“删除用户”

3. 管理内容
   • 管理员在留言墙对所有留言可见“撕掉”按钮
   • 管理员在私信中可删除任何消息

4. 退出管理
   点击“退出管理”按钮恢复普通视图。

修改管理员密码：
   在 index.html 中找到 const ADMIN_PWD = "reset2024"; 替换引号内的值即可。

--------------------------------------------------------------------------------
🧹 数据清空（重置网站）
--------------------------------------------------------------------------------

如需彻底清空所有用户、留言、私信（保留表结构），请在 Supabase SQL Editor 中执行：

TRUNCATE TABLE public_message_replies RESTART IDENTITY CASCADE;
TRUNCATE TABLE public_messages RESTART IDENTITY CASCADE;
TRUNCATE TABLE private_messages RESTART IDENTITY CASCADE;
TRUNCATE TABLE users RESTART IDENTITY CASCADE;

⚠️ 此操作不可逆，请确认无重要数据后执行。
